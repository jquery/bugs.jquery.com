Navbar of *.jquery.com wrapping badly in Chrome stable (Ubuntu 9.10)

Reported by: danbeam Owned by:
Priority: undecided Milestone:
Component: misc Version:
Keywords: browser compatibility, api Cc:
Blocked by: Blocking:
Description

I'm not sure this is where I'd report this bug, but the navbar of jquery.com (and api.jquery.com, dev.jquery.com, etc.) is wrapping the last button to the next line.

I'm using the apt package "google-chrome-stable" from apt in Ubuntu 9.10 (x86_64).

Here's a fix using jQuery to show the difference visually to those with this OS/browser available to them.

$('#jq-secondaryNavigation > ul, #jq-footerNavigation > ul').css('width','50em');

Attached is the patch to the CSS file affected this width rule (http://static.jquery.com/files/rocker/css/screen.css).

I've also attached screenshots of before/after.

Obviously is this breaks things in all other browser, either conditional CSS or some other hackery would need to be employed (and it is up to you guys how important this is to begin with, :P).
