UI-web: Draggable Demos

Reported by: djot Owned by: joern
Priority: major Milestone: 1.2.4
Component: web Version: 1.2.3
Keywords: Cc:
Blocked by: Blocking:
Description

bugs on the ui-website (v1.5b4), occured with FF 2.0.0.14 on WinXP2

______________

0. The bugtracker misses component "UI-web", also Versions for UI

1. Draggables - "Simple image drag" with "Using a helper clone" seems to reload the dragged image from the server instead of just cloning the image. When starting to drag the is only a frame with a "missing image" icon for a second.

Option "Drag and revert to the original position" does not work here also. a) It actually clones the image, the mousepointer is about 50px to the right of the image, reverting does not work really, and the clone gets hidden after dragging, but at a wrong position (50px to the right here also)

2. Draggables - "Drag div element" with option "Drag and revert to the original position" does not drag correctly, is about 50px below the mouse pointer. Also it does not revert to the original position but also about 50px below the original position.

Same for "Using a helper clone" - 50px below mouse pointer.

Option "Cursor position top/left" is not top/left. top is ok, but

left is about 10px outside the box.

Option "Cursor position right/bottom" is inside the box, so bottom is incorrect and right also (about 15px inside the box)

3. Draggables - "Drag prevention for pre-defined elements"

Hm, perhaps I am wrong but this demo there is not a drag prevention demo, is it? It's the same like "Drag with a handle", just wiht the different "select me" part ... but it is draggable the same way like the example above.

djot
