Changed March 14, 2009 12:55AM UTC by dmethvin comment:2

jQuery('#list1, #list2').prepend(li);

Since there are two elements in the jQuery object, the prepend is cloning the original li and appending the clone to the second element. The clone does not have the data.

I think the logic behind this is that copying all the 

.data
to all elements would be expensive (and remember that 
.data
includes all events as well). You still have the option to explicitly copy the data, or better still attach events and data after the implicit cloning occurs.

The docs aren't clear on this, so I'd suggest clearly documenting the behavior to resolve the ticket.

Changed June 02, 2009 10:16AM UTC by mikecapp comment:3

Same issue as #4191, I think. #4284 raises the same issue as an enhancement request.

Whether or not clone (explicit or otherwise) should copy data is a fair question, but it's hard to justify the first element generated by prepend etc getting data and the others not.
